# Data organization вЂ” what to stage before running the pipeline

This file is the **only** contract the pipeline imposes on your data. If
your measurement campaign follows this layout, `run_all` reproduces the
paper's figures and tables verbatim. If your layout differs, edit
`matlab/config/paths.m` (one file) to point each variable at your files.

Nothing below ships. You bring your own measurements.

---

## 2. Raw PDP file naming

Each `<campaign>` processing script globs a specific filename pattern.
Your raw PDP files must match:

| Campaign | Path | File pattern | Count |
|---|---|---|---|
| NYU 142 GHz | `data/raw/nyu_142/` | `142GHz_Outdoor_*.mat` | 27 |
| NYU 6.75 GHz | `data/raw/nyu_7/` | `Data7Pack_TX*_RX*_Aligned.mat` | 18 |
| USC 145 GHz | `data/raw/usc_145/{LoS,NLoS}/` | per-file (see `USC142GHz_Method_Comparison_Full.m`) | 26 |
| USC 6.75 GHz | `data/raw/usc_7/{LOS Study,OLOS Study}/` | per-file (see `USC7GHz_NewData_Processing.m`) | 17 |

The per-campaign README under `share/matlab/processing/<campaign>/`
documents the internal field names each `.mat` file must expose (PDP
arrays, azimuth grid, elevation grid, frequency, TX/RX heights, etc.).

## 3. Antenna patterns & TX-power CSVs

These files are specific to the paper's hardware. You must provide
equivalents for your own measurement gear.

| NYU 6.75 GHz | `share/matlab/processing/nyu_7/7GHz_Outdoor (1).csv` | Per-link TX power table |

## 4. Per-link point tables (`N1/N3/U1/U3`)

The PL-CI-fit pipeline (Fig 5) and Tables N1/N3/U1/U3 read from the
bundled per-link xlsx tables at `data/point_data/`. You have two ways
to populate them:

**(a) Let the raw-processing scripts build them.** `run_all` STEP 1
produces `share/matlab/processing/<campaign>/Results/*.xlsx` which the
paper-figure scripts consume directly. No manual xlsx staging needed.

**(b) Hand-author the xlsx yourself.** If you already have per-link
summary tables from a prior pipeline, deposit them under
`data/point_data/` following the 3-row-header schema described in
`share/matlab/lib/load_point_data.m` (rows: title banner, metric
family, processing variant; first data row at row 4). The variants
recognized are `NYU orig. (N1)`, `NYU thres`, `USC thres`, and
`USC orig. (U1)`. Column families: `TX`, `RX`, `Loc Type`, `TR Sep`,
`Omni PL`, `Omni DS`, `Omni ASA`, `Omni ASD`.

Either way, the paper-figure scripts consume the same schema; option
(a) is recommended so your numbers stay in sync with your raw data.

## 5. Redirecting with `CHANNEL_DATA_ROOT`

If you prefer to keep measurement data outside the repo (e.g. on a
large data volume), set:

```bash
export CHANNEL_DATA_ROOT=/mnt/measurement_drops/myproject/2026/
```

`paths().point_data`, `paths().raw_nyu_142`, etc. then resolve to
`<CHANNEL_DATA_ROOT>/point_data/` and `<CHANNEL_DATA_ROOT>/raw/`
instead of `<repo>/data/`.

Alternatively, edit `matlab/config/paths.m` directly вЂ” it's the single
source of truth for all paths the pipeline touches.

## 6. Minimal sanity check

Before running the full pipeline, confirm staging is correct:

```matlab
P = paths();
assert(isfolder(P.raw_nyu_142),   'NYU 142 raw dir missing');
assert(isfolder(P.raw_nyu_7),     'NYU 7 raw dir missing');
assert(isfolder(P.raw_usc_145),   'USC 145 raw dir missing');
assert(isfolder(P.raw_usc_7),     'USC 7 raw dir missing');
assert(isfile(P.nyu_142_hplane_pattern), 'NYU 142 H-plane pattern missing');
assert(isfile(P.nyu_7_phi0),             'NYU 7 phi0 pattern missing');
assert(isfile(P.usc_145_azicut),         'USC 145 azi cut missing');
assert(isfile(P.usc_7_antenna_pattern),  'USC 7 pattern missing');
fprintf('Staging OK, you can run_all now.\n');
```

If every assertion passes, `run_all` will regenerate the 16 paper
figures from your data in one go.
